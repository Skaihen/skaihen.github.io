Bootstrap

Para empezar con esta guía lo primero es saber de que se trata Bootstrap. Bajo el signo "mobile-first", Bootstrap es un framework orientado a la creación de páginas web adaptables de forma rápida y sencilla, el cual añade elementos como iconos o clases que cuentan con distintos estilos.

Antes de proceder con el resto de la guía cabe mencionar que, pese a las ventajas de Bootstrap a la hora de crear páginas web como la rapidez a corto y largo plazo al tener gran parte de los estilos creados, estos cuentan con la etiqueta !important, y al ser tantos, es posible que nombremos una clase igual a una ya creada por Bootstrap, lo que da pie a errores o creer que nuestro css esta mal y sin saber esto a perder tiempo buscando el fallo. Por suerte son fallos menores que se solucionan cambiando el nombre de nuestra clase o, dependiendo de como implementemos Bootstrap, editables directamente. Cabe destacar que Bootstrap no es necesario para hacer una página, como es el caso de esta, ni para darle estilo, ya que puedes hacer esto con css simplemente si asi lo prefieres.

Instalación y Uso

Bootstrap cuenta con una guía de instalación en su página principal (se abre en una pestaña nueva), a la cual acudiremos más adelante, aunque se resume en tres formas distintas dependiendo de nuestras preferencias.

jsDelivr:
La primera forma y más sencilla es el uso de jsDelivr, con la cual lo unico que debemos hacer es añadir al head de nuestro html un link al css de Bootstrap y una o dos etiquetas script con referencias al javascript de Bootstrap y Popper dependiendo de que elijamos. Ambas etiquetas se pueden encontrar actualizadas en esta sección de su página de descarga (se abre en una pestaña nueva). Esta opción es la más conveniente, ya que tiene toda la funcionalidad de Bootstrap sin complicarnos demasiado y nos es util si no queremos profundizar mucho en la estructura o queremos maquetar una página de forma rápida.
CSS y JS Compilados:
La segunda opción más rápida es descargar el css y js compilados directamente. Esto puede sernos util ya que al tener directamente la hoja de estilos que Bootstrap utiliza, podemos ver como funciona para así aprender a darle un mejor uso o incluso editarla para modificar el comportamiento de las distintas clases. Podemos encontrar el enlace de descarga al principio de la web de descargas (se abre en una pestaña nueva).
Gestor de paquetes:
Por último, podemos usar un gestor de paquetes a la hora de implementar Bootstrap en nuestra aplicación.Esta es la mejor opción en caso de que tengas una aplicación que use otro framework, extensa o compleja, para evitar las posibles incompatibilidades, aunque requiere del uso del compilador Sass y Autoprefixer, los cuales usa Bootstrap para funcionar. En esta página (se abre en una pestaña nueva) se encuentran de forma más detallada que gestores tienen soporte y los comandos a usar.

Breakpoints

Bootstrap está diseñado para ser "responsive", por lo que gran cantidad de sus clases cuentan con la opción del uso de breakpoints. En resumidas cuentas podemos indicar que estilo queremos usar dependiendo del viewpoint, es decir, el tamaño de nuestra pantalla. Por ejemplo, si le damos a un párrafo las clases p-md-2 y p-lg-4, este tendrá un padding de 0.5 rem en pantallas con un ancho mayor o igual a 768px y 1.5 rem en pantallas un ancho mayor o igual a 992px. La estructura básica de estas clases se forma por un prefijo, que indica que estilo queremos aplicar como puede ser "p" para padding o "m" para margin, un interfijo, que denota el tamaño a partir del que funciona nuestra clase, y un sufijo, que sería el tamaño del prefijo de nuestra clase. A continuación dejare una tabla de interfijos para consultar los distintos breakpoints disponibles y otra de sufijos con los distintos tamaños soportados.

Modelo de Caja

Exterior de una caja:

En css todos los objetos tienen una caja alrededor, por así decirlo. Estas interactuan entre si y se usan para alinear objetos. Entender su funcionamiento es clave a la hora de crear el diseño de una página. Lo primero que debes saber es que existen dos tipos de cajas con funciones distintas, las cuales se definen mediante la propiedad "display" y los valores "block" e "inline". Estas se diferencian en lo siguiente:

Cajas "Block":
  • El elemento se posiciona en una línea nueva.
  • La caja se expande en la dirección del inline (representada en la imagen más abajo) y en la mayoria de los casos ocupa todo el espacio disponible en su recipiente.
  • Las propiedades alto y ancho se respetan.
  • El padding, margin y border causan que los otros elementos sean empujados.
  • Algunos elementos HTML como el h1 y el p, usan cajas del tipo "block".
Cajas "Inline":
  • El elemento se posiciona en la misma línea.
  • Las propiedades alto y ancho se ignoran.
  • El padding, margin y border afectan a la caja pero no causan que los otros elementos inline sean empujados.
  • El padding, margin y border horizontal si empujan otras cajas inline.
  • Algunos elementos HTML como el span, em y strong usan cajas inline por defecto.
Interior de una caja:

Las cajas, además de lo mencionado anteriormente, tienen otras propiedades que indican como se van a posicionar los elementos dentro de estas. Por defecto, los elementos dentro de las cajas se comportan normal, es decir, como cualquier otro elemento "block" o "inline".

Podemos cambiar el comportamiento de estos usando la propiedad "display" con valores como grid o flex. En este caso, por ejemplo, si usamos "display: flex" en un elemento, el exterior se comporta como una caja tipo "block", pero el interior cambia a tipo "flex". Cualquier objeto o elemento que se encuentre directamente dentro de este seguira las normas de Flexbox, las cuales están explicadas más adelante.

Modelo Flexbox

Flexbox o Flexible Box es un modulo que provee una forma mucho más sencilla de ordenar, alinear y distribuir el espacio entre los objetos dentro de un contenedor, incluso cuando su tamaño es desconocido y/o dinamico.

La principal idea detras de flex es dar a los objetos dentro del contenedor la abilidad para alterar el alto y ancho de estos para que ocupen el máximo de espacio disponible para que se acomode a todo tipo de pantallas, expandiéndolos o encogiéndolos.

Lo más importante de flexbox es que, a diferencia del tipo block, verticalmente direccional, o el tipo inline, horizontalmente direccional, el tipo flex ignora las direcciones. Mientras los otros funcionan bien para páginas, tienen una gran falta de flexibilidad a la hora de respaldar aplicaciones grandes o complejas, especialmente cuando a cambios de orientación, redimensionar, estirar o encoger se refiere.

Nota: Flexbox es más indicado para los componentes de una aplicación y diseños pequeños, mientras que Grid es más indicado para diseños o layouts a mayor escala.

Ya que flexbox es un módulo completo y no solo una propiedad, involucra varias cosas, incluyendo un set completo de propiedades. Algunas de ellas están hechas para ser colocadas en el contenedor o padre (flex container) mientras que otras van colocadas en los elementos interiores o hijos (flex items). En esta sección de su página (se abre en una pestaña nueva) se encuentra una lista con los distintos atributos aplicables tanto a padres como a hijos con imagenes ilustrativas del funcionamiento de estos.
